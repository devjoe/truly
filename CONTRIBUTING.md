# Contributing

Truly is in Alpha. Please keep contributions small and easy to review.

## Good First Contributions

- Bug reports with clear reproduction steps.
- Documentation fixes.
- Public synthetic fixtures that cover a parser or UI contract.
- Small UI, build, or test fixes.

## Before Large Changes

Open an issue before starting large features, new platform support, permission
changes, or model-provider changes. Alpha priorities may change quickly.

## Private Data

Do not attach real feed screenshots, logged-in page captures, private endpoint
URLs, tokens, or personal browser-profile data to public issues or pull
requests. Reduce bugs to synthetic examples whenever possible.

## Security

Report security issues through `SECURITY.md`. Do not post exploit details,
private data, tokens, or endpoint secrets in public issues.
