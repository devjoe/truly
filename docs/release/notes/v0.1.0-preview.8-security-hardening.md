# Truly 0.1.0 Preview 8 Security Hardening

Status: published to Chrome Web Store as Unlisted
Prepared at: 2026-06-24 CST

## Context

Preview 7 was submitted to Chrome Web Store review before a follow-up security
review flagged development reload behavior and two hardening opportunities. The
next Preview 8 package should include these fixes before replacing the current
CWS draft package.

## Changes

- Production service workers no longer probe `http://localhost:9012/` or accept
  dev-reload signals. `initDevReloadClient()` is now gated behind
  `__TRULY_DEV_BUILD__`.
- `release:preview` runs the shared release bundle audit against both `dist` and
  the packaged extension zip, including checks for dev-reload localhost probes
  and reload markers.
- The MAIN-world to isolated-world SSR cache bridge now validates cache URLs,
  request IDs, post schema, post counts, and serialized payload size before
  reading from or writing to extension storage.
- OpenAI-compatible API keys default to session-only storage. Users can still
  opt into browser-local persistence by unchecking the session-only control.
- Debug snapshot export redacts API-key-shaped fields from sync and session
  storage, and it still avoids reading `chrome.storage.local`.
